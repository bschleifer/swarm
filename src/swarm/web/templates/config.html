One-time setup: Register an app in Microsoft Entra

Swarm reads your email via the Microsoft Graph API using delegated permissions — it can only read mail you explicitly grant access to. No client secret is needed; Swarm uses the PKCE flow (proof key for code exchange).

Already set up in your organization? If someone has already registered a Swarm app in your org’s Entra tenant, you don’t need to create a new one. Ask your admin for the Application (client) ID and Directory (tenant) ID, paste them into the fields below, and click Save & Apply — then skip straight to Connect Microsoft Account.

Step 1 — Create the app registration

  1. Open entra.microsoft.com → App registrations
  2. Click + New registration
  3. Name: Swarm (anything you like)
  4. Leave “Redirect URI” blank for now
  5. Click Register

Step 2 — Add the redirect URI (important: correct platform)

  1. In your new app, go to Authentication in the left sidebar
  2. Click + Add a platform
  3. Choose Mobile and desktop applications
  4. Scroll to Custom redirect URIs at the bottom and enter:
    http://localhost:9090/auth/graph/callback — change 9090 if you use a different port
  5. Click Configure

Step 3 — Grant Mail.Read permission

  1. Go to API permissions in the left sidebar
  2. Click + Add a permission → Microsoft Graph → Delegated permissions
  3. Search for Mail.Read, check it, and click Add permissions
  4. (Org tenants) Click Grant admin consent for [your org] and confirm — green checkmarks should appear next to each permission

Step 4 — Copy your IDs

  1. Go to Overview in the left sidebar
  2. Copy Application (client) ID → paste into the Client ID field below
  3. Copy Directory (tenant) ID → paste into the Tenant ID field below
  4. Click Save & Apply at the top, then click Connect Microsoft Account
Why “Mobile and desktop”? Entra offers three platform types. Web requires a client secret. SPA only allows browser-side token exchange. Mobile and desktop supports PKCE with a server-side callback, which is what Swarm uses. If you accidentally picked the wrong one, go to Authentication, delete it, and add the correct platform.
Users seeing “Need admin approval”? Organization tenants require an Entra admin to grant consent before non-admin users can connect.
  • During setup (recommended): After adding Mail.Read in Step 3, click Grant admin consent for [your org] on the API permissions page — green checkmarks appear and all users are unblocked.
  • After the fact: Go to Entra → Enterprise Applications → find “Swarm” → Permissions → click Grant admin consent for [your org].
Once granted, any user in the organization can connect without the approval prompt.

One-time setup: Register an OAuth app in Atlassian

Swarm connects to Jira via OAuth 2.0 (3LO) — no long-lived API tokens needed. Access tokens refresh automatically and are stored locally.

Step 1 — Create the app

  1. Open developer.atlassian.com → My Apps
  2. Click Create → OAuth 2.0 integration
  3. Name: Swarm

Step 2 — Configure permissions

  1. Go to Permissions, click Add for Jira API
  2. Add scopes: read:jira-work, write:jira-work

Step 3 — Set callback URL

  1. Go to Authorization → OAuth 2.0 (3LO)
  2. Callback URL: {% if config.domain %}https://{{ config.domain }}{% else %}http://localhost:{{ config.port }}{% endif %}/auth/jira/callback

Step 4 — Copy credentials

  1. Go to Settings
  2. Copy Client ID and Secret into the fields below
  3. Click Save & Apply, then Connect Jira
